# Security Policy

## Supported Versions

| Version | Supported |
| --- | --- |
| `master` (latest) | Yes |
| older branches/tags | No |

## Reporting a Vulnerability

If you discover a security issue:

1. Do not disclose it publicly in Issues or Discussions.
2. Open a private security advisory in GitHub (preferred).
3. If advisory flow is unavailable, open an issue with minimal details and request private contact.

Include:

- affected endpoint(s)
- reproduction steps
- impact assessment
- proposed mitigation (if available)

## Response Expectations

- Initial triage target: within 5 business days
- Remediation timeline: based on severity and exploitability
- Credit: provided upon request after a fix is published
