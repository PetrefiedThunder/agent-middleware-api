# Agent-Native Middleware API — LLM-Readable Documentation

**Version:** 0.4.1
**Base URL:** http://localhost:8000
**Auth:** X-API-Key header

---

## What This API Does

The Agent-Native Middleware API provides infrastructure for autonomous AI agents:
- **Billing & Payments** — Credit-based wallet system with Stripe integration
- **Telemetry** — Event tracking and anomaly detection
- **Agent Communication** — Agent-to-agent messaging and swarm coordination
- **AI Decision Making** — Autonomous decisions, self-healing, memory
- **MCP Tools** — Model Context Protocol for tool discovery and execution
- **AWI** — Agentic Web Interface for web automation with human pause/steer
- **Sandbox** — Safe testing environments (dry-run and behavioral)

---

## Quick Start for Agents

### 1. Discover What This API Offers

```bash
GET /v1/discover
```

Returns all capabilities, MCP tools, AWI endpoints, and pricing.

### 2. Get Your API Key

Contact the service operator. Pass it as `X-API-Key: your-key` on every request.

### 3. Create a Wallet

```bash
POST /v1/billing/wallets/agent
{
  "wallet_id": "your-agent-id",
  "parent_wallet_id": "sponsor-001",
  "max_spend": 1000
}
```

### 4. Start Using Services

```bash
# Emit telemetry
POST /v1/telemetry/events
{"event": "agent_started", "agent_id": "your-agent-id"}

# Send message to another agent
POST /v1/comms/messages
{"from": "your-agent-id", "to": "other-agent", "content": "hello"}

# Make AI decision
POST /v1/ai/decide
{"agent_id": "your-agent-id", "context": {...}, "options": [...]}
```

## MCP Tool Discovery

Agents using Model Context Protocol should:

```bash
# List available tools
GET /mcp/tools.json

# Call a tool via MCP
POST /mcp/messages
{"jsonrpc": "2.0", "method": "tools/list", "id": 1}
```

Available tools:
- `emit_telemetry_event` — Track events (1 credit)
- `charge_wallet` — Deduct credits (1 credit)
- `send_agent_message` — Send message (1 credit)
- `decide` — AI decision (10 credits)
- `heal` — Self-healing (15 credits)
- `create_session` — AWI session (5 credits)

## AWI — Agentic Web Interface

For web automation without fighting DOM trees:

```bash
# Create session
POST /v1/awi/sessions
{"target_url": "https://example.com", "max_steps": 100}

# Execute standardized actions
POST /v1/awi/execute
{"session_id": "...", "action": "search_and_filter", "parameters": {...}}

# Get representation
POST /v1/awi/represent
{"session_id": "...", "representation_type": "summary"}

# Human pause/steer
POST /v1/awi/intervene
{"session_id": "...", "action": "pause", "reason": "Review needed"}
```

### AWI Phase 9 — Enhanced Capabilities

**Passkey Authentication** — FIDO2/WebAuthn passkey support for agent sessions:

```bash
# Register a passkey for agent authentication
POST /v1/awi/passkey/register
{"wallet_id": "agent-001", "rp_name": "MyAgent", "user_name": "agent@example.com"}

# Generate passkey challenge
POST /v1/awi/passkey/challenge
{"wallet_id": "agent-001"}

# Verify passkey assertion
POST /v1/awi/passkey/verify
{"wallet_id": "agent-001", "assertion": {...}}

# List registered passkeys
GET /v1/awi/passkey/list/{wallet_id}

# Delete a passkey
DELETE /v1/awi/passkey/{credential_id}
```

**DOM Bridge** — Bidirectional DOM↔AWI translation for precise web control:

```bash
# Get DOM snapshot as structured data
POST /v1/awi/dom/snapshot
{"session_id": "...", "include_shadow_dom": true}

# Get element at position (clickable elements)
POST /v1/awi/dom/element_at
{"session_id": "...", "x": 100, "y": 200}

# Execute AWI action and return DOM result
POST /v1/awi/dom/execute
{"session_id": "...", "action": "click", "parameters": {"element_id": "btn-submit"}}

# Query DOM with CSS selector
POST /v1/awi/dom/query
{"session_id": "...", "selector": "button.primary"}

# Generate AWI action from DOM element
POST /v1/awi/dom/to_awi
{"session_id": "...", "element": {...}}
```

**RAG Memory** — Vector store for semantic agent memory:

```bash
# Ingest document into memory store
POST /v1/awi/rag/ingest
{"wallet_id": "agent-001", "content": "...", "metadata": {"source": "doc.pdf"}}

# Search memory semantically
POST /v1/awi/rag/search
{"wallet_id": "agent-001", "query": "what was the previous checkout flow?", "top_k": 5}

# Get memory context for session
POST /v1/awi/rag/context
{"wallet_id": "agent-001", "session_id": "..."}

# List memory entries
GET /v1/awi/rag/list/{wallet_id}

# Delete memory entry
DELETE /v1/awi/rag/{memory_id}

# Clear all memory for wallet
DELETE /v1/awi/rag/clear/{wallet_id}
```

## Billing & Pricing

### Credit System
- 1 credit = $0.001 USD
- Free tier: 1000 credits/month
- Purchase credits via Stripe at `/v1/billing/top-up`

### Rate Limits
- 120 requests/minute per API key
- Burst allowance: 20 extra

### Velocity Monitoring
Wallets are auto-frozen if spending exceeds:
- 1000 credits/hour
- 10000 credits/day
